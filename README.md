# Madi Manager

Madi Manager — локальное приложение для управления проектами, объектами, сотрудниками и табелем рабочего времени. Цель проекта — заменить Excel/Power BI для внутренних задач, сохранив при этом контроль над данными и работу без интернет‑подключения. Приложение построено на **React**, **TypeScript** и **Vite** для фронтенда и **Node/Express** с **better‑sqlite3** для хранения данных в реальном файле SQLite.

## Возможности

- **Дашборд.** Сводные показатели (количество сотрудников, объектов, назначений, записей табеля, дублей) визуализируются в виде графика.
- **Импорт Excel/CSV.** Выберите тип данных (сотрудники, объекты, табель), загрузите файл, просмотрите предварительные данные и импортируйте их в базу. При импорте происходит проверка дублей — существующие записи логируются в таблице `DuplicateLog`.
- **Дубли.** Отдельная страница показывает журнал дублей с типом, действием, хешем строки и датой.
- **Табель рабочего времени.** Отображает сотрудников строками и дни месяца колонками. Есть фильтры по месяцу, объекту и бригаде, массовая отметка присутствия и экспорт табеля в Excel.
- **Сотрудники.** CRUD‑интерфейс для добавления, отключения и просмотра сотрудников.
- **Объекты.** CRUD‑интерфейс для рабочих объектов (название, адрес, тип, статус, проект, ответственный, комментарий).
- **Распределение.** Назначение сотрудников на объекты с возможностью выбрать несколько сотрудников и назначить их на объект, а также снять назначение.
- **Командный центр.** Основной экран управления отображает готовность объектов, блокеры, переработки, последние импорты и дубли.
- **Экспорт.** Позволяет выгружать из базы данные о сотрудниках, объектах, табеле или дублях в формате Excel.

## Архитектура

- **Frontend.** Разработан на React + TypeScript + Vite. UI строится на Material UI и Tailwind CSS. Таблицы реализованы через TanStack Table, графики — через Recharts. Весь код находится в каталоге `src/`, каждый экран вынесен в модуль (`modules/*`), общие компоненты — в `components/`.
- **Backend.** В каталоге `server/` размещён простой Express‑сервер, который инициализирует SQLite‑базу в файле `data/madi-manager.db` (создаётся автоматически) и предоставляет REST‑API: `/api/employees`, `/api/objects`, `/api/assignments`, `/api/timesheet`, `/api/duplicates`, `/api/import`, `/api/export`, `/api/dashboard`, `/api/command-summary`, `/api/health`. База данных инициализируется согласно `src/db/schema.sql`.
- **База данных.** Используется файл SQLite с таблицами `Templates`, `Projects`, `EntityNodes`, `MetricsData`, `ImportBatches`, `DuplicateLog`, `Employees`, `WorkObjects`, `WorkAssignments`, `Timesheet`, `AppSettings`. Данные сохраняются между запусками.
- **Тестирование.** Для юнит‑тестов используется Vitest. Пример теста проверяет корректность маршрутизации и отображения заголовков.
- **CI/CD.** В директории `.github/workflows/` находится файл `ci.yml`, который запускает GitHub Actions: установка зависимостей (`npm ci`), сборка фронтенда (`npm run build`), тесты (`npm test -- --run`), проверка типов (`npx tsc --noEmit`), аудит (`npm audit --audit-level=high`) и загрузка артефакта сборки (`dist`).

## Запуск проекта

Для локального запуска выполните:

```bash
npm install
npm run dev
```

Команда `npm run dev` запускает одновременно Vite (`localhost:5173`) и API‑сервер (`localhost:5174`). После сборки (`npm run build`) статические файлы попадают в каталог `dist/`.

Для проверки типов и запуска тестов:

```bash
npm test -- --run
npx tsc --noEmit
npm audit --audit-level=high
```

## Организация кода

- `src/App.tsx` — корневой компонент приложения с маршрутизацией и боковым меню.
- `src/components/Sidebar.tsx` — навигационное меню.
- `src/modules/` — модули интерфейса: `dashboard`, `import`, `duplicates`, `timesheet`, `employees`, `objects`, `assignments`, `command-center`, `export`, `settings`.
- `src/db/schema.sql` — SQL‑схема базы.
- `server/index.js` — API‑сервер на Express с использованием better‑sqlite3.
- `.github/workflows/ci.yml` — конфигурация CI.

## Desktop‑версия

Функциональность приложения спроектирована так, чтобы в перспективе собрать полноценную десктоп‑версию. Для этого можно использовать Tauri или Electron с подключением к тем же таблицам SQLite. Исследование подходящих Tauri‑плагинов для работы с SQLite приведено в `GITHUB_RESEARCH.md`. На текущем этапе desktop‑сборка не включена, но код backend из каталога `server/` можно адаптировать в Tauri или Electron.
